🎨 Heroku CLI Color Palette Demo

This demo showcases all the colors defined in the new Heroku CLI design system.


📱 App-Related Colors:

  app - Name of an app (147, purple)

  pipeline - Name of a pipeline (uses magenta color, #FF22DD)

  space - Name of a space (117, blue)

  datastore - Name of a heroku datastore (185, yellow)

  addon - Name of an addon (185, yellow)

  attachment - Name of an attachment (uses gold color, ANSI256: 220)

  name - Name of heroku entity without special color (uses pink color, ANSI256: 212)


📊 Status Colors:

  success - Success messages and states (40, green)

  failure - Failure, error messages and states (#FF8787, red)

  warning - Warning messages (214, orange)

  inactive - Disabled and unknown states (248, gray)


👥 User/Team Colors:

  team - Heroku team/org (uses light cyan, #8FF5F7)

  user - Heroku user/email (#50D3D5, cyan)


🔧 General Purpose Colors:

  label - Labels, table headers, keys (default, bright white/black, bold)

  info - Help text, soft alerts (43, teal)

   $ heroku command --flag  - Command examples and code blocks (255, white on dark gray)

🔀 Mixing Custom and ANSIS colors:

You can combine custom Heroku colors with ansis functionality for more complex styling.


// Using ansis with custom colors

  [1mmy-app[22m      // Bold custom app color

  [4mSuccess![24m // Underlined success

  [44mmy-team[49m  // Team name with blue background


// Using custom colors with ansis styling

  Deploying my-app to production ✓

  Error: [1mFailed to connect to[22m DATABASE

  Warning: [3mThis will add charges to your account[23m


🌈 ANSIS Functionality:

The color interface also provides direct access to ansis functionality for additional styling options.


// Text styling

  [1mbold text[22m               // Bold

  [3mitalic text[23m           // Italic

  [4munderlined text[24m    // Underlined


// Combined styling

  [1m[31mbold red[39m[22m            // Bold + red

  [4m[34munderlined blue[39m[24m // Underlined + blue

  [1m[3mbold italic cyan[23m[22m // Multiple styles


// Custom colors

  [38;2;255;107;107mcustom hex color[39m    // Custom hex color

  [38;2;255;107;107mRGB color[39m   // RGB color


🎯 Color Palette Summary:

The color interface provides two complementary approaches:


  [1mCustom Heroku colors:[22m

    • Semantic functions (app, success, etc.)

    • Purpose-built for Heroku CLI consistency

    • Include icons and specific styling

    • Use design system colors (147, 40, etc.)


  [1mANSIS Colors:[22m

    • Direct access to all ansis functionality

    • Standard ANSI colors ([31mred[39m, [34mblue[39m, etc.)

    • Text styling ([1mbold[22m, [3mitalic[23m, etc.)

    • Custom colors ([38;2;255;107;107mhex[39m, [38;2;255;107;107mRGB[39m)

    • Chainable methods for complex styling


All colors are designed to be accessible and consistent across the Heroku CLI experience.

Status colors follow semantic conventions: green for success, red for errors, orange for warnings.

Purple is used for apps, magenta for pipelines, pink for general names, yellow for addons and datastores, and gold for attachments.

Both approaches can be used together for maximum flexibility while maintaining Heroku CLI design consistency.
